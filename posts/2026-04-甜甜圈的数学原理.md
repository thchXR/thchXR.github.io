此笔记为著名的旋转甜甜圈程序及其背后的数学原理

## 数学原理
我们在这里先介绍程序背后的数学原理，我们的目的是在屏幕上绘制一个旋转的甜甜圈图像，首先我们要先了解一下3D渲染的基本原理

![示意图](/src/images/2026-4/perspective.png)


如上图所示，我们想把三维空间中物体表面想象的任意点 $(x,y,z)$ 投影到屏幕上的一点 $(x',y')$ 上，由于上图的视角是从侧面出发的，所以只能看到 $y$ 和 $z$ 轴，实际上还要考虑 $x$ 轴。

对于图像的绘制过程可以分为三个步骤，第一步为计算三维空间中想象的物体的坐标 $(x,y,z)$ ，第二步为把想象的物体映射到屏幕上的 $(x',y')$ ，第三步计算关照强度与阴影深度

我们先考虑第一步，即如何绘制一个环面（torus），显然环面是一个旋转体，我们做出其截面图如下

![torusxsec](/src/images/2026-4/torusxsec.png)

我们用角度 $\theta \in [0,2\pi]$ 以极坐标的方式表示此截面上的点 $(x,y,z)$
$$
(x,y,z) = (R_2+R_1\cos\theta,R_1\sin\theta,0)
$$
但如上的点仅是在单一截面上的点，我们需要在其基础上再乘以一个旋转矩阵 $R_y(\phi)$ 其中 $\phi \in [0,2\pi]$ ，计算过程如下
$$
(R_2+R_1\cos\theta,R_1\sin\theta,0)\cdot 
\begin{bmatrix}
\cos\phi & 0 & \sin\phi\cr
0 & 1 & 0\cr
-\sin\phi & 0 & \cos\phi
\end{bmatrix}
$$
$$
= ((R_2 + R_1 \cos\theta)\cos\phi, R_1\sin\theta,(R_2+R_1\cos\theta)\sin\phi)
$$

通过如上过程，我们使用角度 $\theta,\phi$ 可以表示出静止的环面上任意点的坐标。我们现在还希望环面能够绕着 $x$ 轴和 $z$ 轴进行旋转，所以我们在上述结果的基础上再乘上两个旋转矩阵 $R_x(A), R_z(B)$ ，我暂时把结果写为
$$
(x,y,z)=(R_2+R_1\cos\theta,R_1\sin\theta,0) \cdot R_y(\phi) \cdot R_x(A) \cdot R_z(B)
$$

接下来考虑第二步，根据上面投影的示意图，显然我们可以得到以下等式
$$
\begin{aligned}
x' &= \frac{x z'}{z}\cr
y' &= \frac{y z'}{z}
\end{aligned}
$$

由于 $z'$ 实际上只影响图像在屏幕上呈现的大小，所以可以视为常数，设为 $K_1$ ，则在屏幕上的图像坐标 $(x',y')$ 可以写为 $(\frac{x K_1}{z},\frac{y K_1}{z})$ ，我们可以根据我们想呈现的图像大小随意选择 $K_1$ 的大小

但我们在上述映射过程中，我们可能把不同的点 $(x,y,z)$ 绘制到屏幕上同一点 $(x',y')$ ，我们可以使用深度缓冲以防止这种情况出现（具体应用见代码部分）

接下来我们可以把第一步计算出的坐标 $(x,y,z)$ 代入第二步中，但先等一下，注意到我们现在的环面是在绕着原点旋转的，我们现在要把环面移动到屏幕之后，所以对坐标做一下平移变换，得到
$$
\begin{aligned}
x' &= \frac{K_1 x}{K_2 + z}\cr
y' &= \frac{K_1 y}{K_2 + z}
\end{aligned}
$$

现在我们就可以把上面计算出的结果代入了
$$
\begin{pmatrix}
	x\cr y\cr z\cr
\end{pmatrix}
=
\begin{pmatrix}
	(R_2 + R_1\cos\theta)(\cos B\cos\phi + \sin A\sin B\sin\phi) - R_1\cos A\sin B\sin\theta\cr 
	(R_2 + R_1\cos\theta)(\cos\phi\sin B - \cos B\sin A\sin\phi) + R_1\cos A\cos B\sin\theta\cr 
	\cos A (R_2 + R_1\cos\theta)\sin\phi + R_1\sin A \sin\theta\cr
\end{pmatrix}
$$

最后我们来考虑第三步，我们使用在计算机图形学中常用的漫反射光的计算方法，即计算法线向量和光照方向的点积，若点积为正则该点表面有漫反射，为负则没有。我们先计算法线向量，显然对于上述过程中取得的截面上的点 $(R_2+R_1\cos\theta,R_1\sin\theta,0)$ 其法线向量显然为 $(\cos\theta,\sin\theta,0)$ 我们在此基础上再乘以三个旋转矩阵以算出法线向量
$$
(N_x,N_y,N_z) = (\cos\theta,\sin\theta,0)\cdot R_y(\phi) \cdot R_x(A) \cdot R_z(B)
$$

接下来我们只需选择我们想要的光照方向向量即可计算出任意点上的关照强度，在这里我们将把光源设置在观测者后上方，即 $(0,1,-1)$ 为光照向量。（注：在高等代数上我们一般要把该光照向量规范化，不过在这里我们可以直接使用 $(0,1,-1)$ 这一点将在代码部分解释）我们计算出的光照强度如下
$$
\begin{aligned}
	L &= (N_x,N_y,N_z) \cdot (0,1,-1)\cr
	&= \cos\phi\cos\theta\sin B - \cos A\cos\theta\sin\phi-\sin A\sin\theta+\cos B(\cos A\sin\theta-\cos\theta\sin A\sin\phi)
\end{aligned}
$$

到此为止此程序背后的数学原理已经基本解释清楚了，我们只需确定几个常数 $R_1,R_2,K_1,K_2$ 的数值就可以代入计算了

## 代码

以下是 `donut.c` 的代码，我们接下来逐步解释这段代码
```
#include <stdio.h>
#include <math.h>
#include <stdint.h>
#include <string.h>
#include <unistd.h>

int main() {
    float A = 0, B = 0;
    float i, j;
    int k;
    float z[1760];
    char b[1760];
    printf("\x1b[2J");
    for(;;) {
        memset(b,32,1760);
        memset(z,0,7040);
        for(j=0; j < 6.28; j += 0.07) {
            for(i=0; i < 6.28; i += 0.02) {
                float c = sin(i);
                float d = cos(j);
                float e = sin(A);
                float f = sin(j);
                float g = cos(A);
                float h = d + 2;
                float D = 1 / (c * h * e + f * g + 5);
                float l = cos(i);
                float m = cos(B);
                float n = sin(B);
                float t = c * h * g - f * e;
                int x = 40 + 30 * D * (l * h * m - t * n);
                int y= 12 + 15 * D * (l * h * n + t * m);
                int o = x + 80 * y;
                int N = 8 * ((f * e - c * d * g) * m - c * d * e - f * g - l * d * n);
                if(22 > y && y > 0 && x > 0 && 80 > x && D > z[o]) {
                    z[o] = D;
                    b[o] = ".,-~:;=!*#$@"[N > 0 ? N : 0];
                }
            }
        }
        printf("\x1b[H");
        for(k = 0; k < 1761; k++) {
            putchar(k % 80 ? b[k] : 10);
            A += 0.00004;
            B += 0.00002;
        }
        usleep(30000);
    }
    return 0;
}
```
