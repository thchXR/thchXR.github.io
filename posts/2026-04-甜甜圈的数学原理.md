# 甜甜圈的数学原理
此笔记为著名的旋转甜甜圈程序及其背后的数学原理

## 数学原理
我们在这里先介绍程序背后的数学原理，我们的目的是在屏幕上绘制一个旋转的甜甜圈图像，首先我们要先了解一下3D渲染的基本原理

![示意图](/src/images/2026-4/perspective.png)

如上图所示，我们想把三维空间中想象的点 $(x,y,z)$ 投影到屏幕上的一点 $(x',y')$ 上，由于上图的视角是从侧面出发的，所以只能看到 $y$ 和 $z$ 轴。

对于图像的绘制过程可以分为两个步骤，第一步为计算三维空间中想象的物体的坐标 $(x,y,z)$ ，第二步为把想象的物体映射到屏幕上的 $(x',y')$

我们先考虑第二步，显然我们可以得到以下等式
$$
\begin{aligned}
x' &= \frac{x z'}{z}\\
y' &= \frac{y z'}{z}
\end{aligned}
$$

由于 $z'$ 实际上只影响图像在屏幕上呈现的大小，所以可以视为常数，设为 $K_1$ ，则在屏幕上的图像坐标 $(x',y')$ 可以写为 $(\frac{x K_1}{z},\frac{y K_1}{z})$ ，我们可以根据我们想呈现的图像大小随意选择 $K_1$ 的大小

但我们在上述映射过程中，可能把不同的点 $(x,y,z)$ 绘制到屏幕上同一点 $(x',y')$ ，我们可以使用深度缓冲以防止这种情况出现（具体应用见代码部分）

接下来我们考虑第一步，即如何绘制一个环形

## 代码

以下是 `donut.c` 的代码
```
#include <stdio.h>
#include <string.h>
#include <math.h>
#include <unistd.h>

#define HIDE_CURSOR() printf("\033[?25l")

int main() {
    float A = 0, B = 0;
    float i, j;
    int k;
    float z[1760];
    char b[1760];
    HIDE_CURSOR();
    printf("\x1b[2J");
    for(;;) {
        memset(b,32,1760);
        memset(z,0,7040);
        for(j=0; j < 6.28; j += 0.07) {
            for(i=0; i < 6.28; i += 0.02) {
                float c = sin(i), d = cos(j), e = sin(A), f = sin(j), g = cos(A);
                float h = d + 2, D = 1 / (c * h * e + f * g + 5), l = cos(i);
                float m = cos(B), n = sin(B), t = c * h * g - f * e;
                int x = 40 + 30 * D * (l * h * m - t * n);
                int y= 12 + 15 * D * (l * h * n + t * m);
                int o = x + 80 * y;
                int N = 8 * ((f * e - c * d * g) * m - c * d * e - f * g - l * d * n);
                if(22 > y && y > 0 && x > 0 && 80 > x && D > z[o]) {
                    z[o] = D;
                    b[o] = ".,-~:;=!*#$@"[N > 0 ? N : 0];
                }
            }
        }
        HIDE_CURSOR();
        printf("\x1b[H");
        for(k = 0; k < 1761; k++) {
            putchar(k % 80 ? b[k] : 10);
            A += 0.00004;
            B += 0.00002;
        }
        usleep(30000);
    }
    return 0;
}
```
